# Story 01-03: 人格设置界面

> Epic: Epic-01 基础聊天界面
> Priority: Medium
> Points: 3

## 概述

实现 AI 人格设置功能。用户可以从预设人格中选择，或配置基础人格参数（如名称、性格描述），为后续的角色扮演功能奠定基础。

## 用户故事

### US-1: 选择预设人格
**作为** 用户，
**我希望** 能够从预设的 AI 人格列表中选择一个，
**以便** 快速切换不同风格的对话体验。

### US-2: 查看人格信息
**作为** 用户，
**我希望** 能够查看当前人格的详细信息，
**以便** 了解 AI 的角色设定。

### US-3: 配置人格 Prompt
**作为** 用户，
**我希望** 能够配置人格的 System Prompt，
**以便** 更加精准地定义 AI 的行为和语气。

## 功能范围

### 包含
- 人格设置入口（侧边栏或聊天区域）
- 预设人格列表展示
- 人格选择和保存
- 人格 System Prompt 配置
- 当前人格状态显示
- 人格数据持久化

### 已实现（UI 层面）
- ✅ 设置按钮入口（`Sidebar.vue` 底部的 "设置" 按钮）

### 不包含（后续迭代）
- 自定义人格创建
- 人格导入/导出
- 人格头像上传
- 人格关联多模态设置

## 验收标准

- [ ] AC-1: 点击设置按钮可打开人格设置界面
- [ ] AC-2: 界面展示至少 3 个预设人格选项
- [ ] AC-3: 选择人格后在聊天界面可见当前人格标识
- [ ] AC-4: 刷新页面后人格选择保持不变
- [ ] AC-5: 用户可以编辑并保存人格的 System Prompt

## 依赖关系

### 前置依赖
- Story 01-02: 会话管理（人格可与会话关联）

### 被依赖（后续）
- 自定义角色创建功能（Epic-02+）

## 备注

- 可使用 Dialog 或 Drawer 组件展示设置界面
- 预设人格数据可硬编码在前端
- 人格设置影响的是 System Prompt，当前仅做 UI 展示和存储
