# Story 01-04: LLM 配置界面

> Epic: Epic-01 基础聊天界面
> Priority: Medium
> Points: 3

## 概述

实现 LLM（大语言模型）配置功能。用户可以配置 API 代理地址、API 密钥和模型名称，为后续接入真实 AI API 做准备。配置界面将采用左右布局，左侧为设置项列表，右侧为具体配置详情。

## 用户故事

### US-1: 配置 LLM 参数
**作为** 用户，
**我希望** 能够配置 API 代理地址 (Base URL)、API 密钥 (API Key) 和模型名称 (Model Name)，
**以便** 能够灵活连接到不同的 LLM 服务提供商。

### US-2: 设置界面导航
**作为** 用户，
**我希望** 点击主页面左下角的“设置”按钮进入一个独立的设置界面，
**以便** 在结构清晰的界面中管理应用配置。

## 功能范围

### 包含
- **设置入口**：主页面左下角添加“设置”按钮。
- **设置界面布局**：
    - 左侧：设置列表（首项为“LLM 设置”）。
    - 右侧：设置详情区域。
- **配置字段**：
    - API Base URL (代理地址)
    - API Key (密钥)
    - Model Name (模型名称)
- **约束**：目前仅支持配置一个 LLM 实例，不支持多模型切换。
- **配置数据持久化**：保存在前端持久化存储中（如 Pinia + LocalStorage）。

### 已实现（UI 层面）
- ✅ 模型选择器占位 UI（`ChatArea.vue` 输入框左下角的模型显示，需联动配置的模型名称）

### 不包含（后续迭代）
- API 密钥有效性验证
- 多 LLM 配置切换
- 高级参数配置（Temperature、Max Tokens 等）
- 使用量统计
